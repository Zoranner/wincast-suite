# 客户端全屏启动体验计划

## 目标

Linux 客户端从“双击启动后等待终端输出”收敛为“全屏窗口立即启动、显示加载进度、首帧到达后直接进入远程画面、宿主程序退出后客户端自动退出”。

## 设计原则

- 不引入 Slint。当前客户端已经使用 SDL2 负责视频窗口和输入事件，加载页也由同一个 SDL 窗口承载，避免两个 GUI 事件循环并存。
- 不增加配置界面、主机列表、日志面板等额外功能。客户端只负责启动、等待、显示、输入、退出。
- 宿主端已有 `ProgramExited` 协议错误和 `Goodbye` 收尾消息，客户端应将其识别为正常会话结束。
- 关闭客户端窗口仍发送 `StopSession`，保持 Linux 端退出时 Windows 端清理本次启动程序树。

## 代码边界

- `crates/wincast-client/src/runtime.rs`：增加客户端状态事件，运行时在读取配置、连接、握手、等待会话、等待首帧、运行结束时上报状态。
- `crates/wincast-client/src/stream.rs`：区分视频流错误和正常会话结束；`ProgramExited` 与 `Goodbye` 走正常结束路径。
- `crates/wincast-render/src/lib.rs`：扩展渲染配置，支持窗口模式和加载状态抽象。
- `crates/wincast-render/src/sdl.rs`：Linux SDL 窗口启动即创建全屏窗口，先绘制假的加载进度条，首帧后复用窗口渲染视频。
- `crates/wincast-client/src/main.rs` 和 `src/lib.rs`：默认入口调用新的全屏 GUI 路径；测试仍可调用协议/边界验证路径。
- `README.md`、`docs/current-status.md`、`docs/smoke-test.md`：同步说明客户端启动体验和宿主程序退出行为。

## 实施步骤

- 为 `ProgramExited` 编写客户端回归测试：宿主端在会话期间发送 `Error { code: ProgramExited }` 后 `Goodbye`，客户端应返回正常结束消息而不是失败或重试。
- 实现会话结束类型，让 `read_h264_encoded_frames_*` 能返回“宿主程序退出”“宿主正常 Goodbye”“视频中断”等不同结果。
- 为运行时状态事件编写单元测试，验证连接、握手、等待首帧、进入视频流等状态按顺序上报。
- 改造运行时 API，让 CLI 和 GUI 都使用同一套连接流程，只替换状态 reporter 和渲染器创建策略。
- 为 SDL 渲染器增加加载状态渲染接口，先用简单几何绘制进度条，不依赖字体库。
- 将 Linux 默认客户端改成全屏 SDL 路径；非 Linux 仍保留协议边界验证路径，避免本机 Windows 开发环境误触 Linux-only SDL。
- 更新文档，并运行项目规定验证。
