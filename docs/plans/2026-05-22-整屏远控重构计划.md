# 整屏远控重构计划

## 目标

将项目主链路从“窗口优先、显示器兜底”重构为“启动指定程序后捕获整块屏幕”。稳定版只面向单显示器、Windows 10 1809 / Build 17763 宿主端和 Linux 客户端远控，不再维护窗口捕获、捕获模式选择和窗口标题定位。

## 架构边界

- `wincast-protocol` 只描述配置、协议消息和输入事件，不包含窗口捕获概念。
- `wincast-host` 负责会话生命周期：接收 Linux 客户端连接、启动配置程序、延迟、启动整屏捕获、推流、读取输入、会话结束后清理程序。
- `wincast-capture` 只暴露整屏捕获 API，Windows 实现使用 DXGI Desktop Duplication。
- `wincast-input` 只按整屏捕获区域映射输入，调用方不再区分窗口坐标和显示器坐标。
- `wincast-media`、`wincast-render` 和 `wincast-client` 保持现有职责，除配置文档同步外不做架构迁移。

## 配置模型

新的 host 配置结构：

```toml
listen = "0.0.0.0:7856"

[program]
path = 'C:\Windows\System32\notepad.exe'
args = []
work_dir = 'C:\Windows\System32'
startup_delay_ms = 3000

[video]
width = 1280
height = 720
fps = 30
codec = "h264"
bitrate_kbps = 4000
max_bitrate_kbps = 6000

[capture]
first_frame_timeout_ms = 5000
```

`startup_delay_ms` 是启动程序后的固定等待时间，允许为 0。`first_frame_timeout_ms` 是捕获首帧保护超时，必须大于 0。旧的 `mode`、`window_title_contains` 和 `startup_timeout_ms` 不再兼容。

## 实施步骤

1. 重构配置模型和示例配置。
   - 修改 `crates/wincast-protocol/src/config.rs`，新增 `ProgramConfig`，删除 `CaptureMode` 和窗口字段。
   - 更新 `crates/wincast-protocol/tests/config.rs`、`examples/host.toml`、README 配置示例。

2. 收缩 capture 公共 API。
   - 将 `CaptureTarget` 改为整屏目标，或提供 `CaptureSession::start_screen()`。
   - 删除稳定版公共模型中的窗口目标字段。
   - Windows 实现只保留 DXGI Desktop Duplication 主链路，移除 WGC/window interop 主路径。

3. 重构 host 会话主链路。
   - 移除 `WindowLocator` 对 `handle_control_client`、listener runtime 和 `run_started_session` 的依赖。
   - 启动程序后执行 `program.startup_delay_ms`。
   - 捕获启动只请求整屏。
   - 输入 bounds 来自捕获帧和屏幕原点，不再通过窗口 rect 计算。

4. 清理测试和模块结构。
   - 删除窗口定位相关会话测试，新增整屏捕获、启动延迟、提前失败清理测试。
   - 主链路完全不再依赖窗口定位后删除 `window.rs`。
   - 将 host agent 层职责重新命名或拆分，使 capture lifecycle 和 stream lifecycle 更清楚。

5. 同步文档。
   - 更新 `docs/design.md`、`docs/development.md`、`docs/deployment-prep.md`、`docs/smoke-test.md`、`docs/current-status.md`。
   - 文档口径改为“单显示器整屏远控”，不再写窗口捕获兜底。

6. 验证和提交。
   - 执行 `cargo fmt --all`。
   - 执行 `cargo test --workspace --all-targets --all-features`。
   - 执行 `cargo clippy --workspace --all-targets --all-features -- -D warnings`。
   - 若变更形成完整逻辑单元，按仓库中文提交风格提交。

## 工程结构检查结论

当前 workspace 的 crate 划分基本合理，`protocol/capture/media/render/input/host/client` 边界清楚。本次重构已处理的结构问题集中在两处：

- `wincast-host/src/agent/capture.rs` 删除窗口定位和捕获目标选择职责，只保留整屏捕获启动与输入 bounds 计算。
- `wincast-capture` 公共 API 删除窗口捕获目标，只暴露 `CaptureTarget::Screen`。

后续若继续提升结构质量，重点应放在 host agent 层的会话流、推流流和输入流命名边界，而不是恢复窗口捕获分支。
